简介

Database Rider通过JUnit规则继承了JUnit和DBUnit，对于基于CDI的测试，还集成了CDI拦截器。这种强大的组合使您可以轻松地准备数据库状态以通过yaml，xml，json，xls或csv文件进行测试。

即将出现的示例的源代码可以在以下github上找到：https://github.com/database-rider/getting-started

原文地址：https://database-rider.github.io/getting-started/#introduction

配置项

有两种配置类型：DataSet和DBUnit

DataSet

名称 释义 默认
value 数据集的文件名。多个文件之前用逗号隔开。
executorId 给定数据集的数据集执行程序的名称。 DataSetExecutorImpl.DEFAULT_EXECUTOR_ID
strategy CLEAN_INSERT, INSERT, REFRESH 和UPDATE。 CLEAN_INSERT，这意味着DBUnit将先清理然后在提供的数据集上存在的表中插入数据。
useSequenceFiltering 如果为true，dbunit将查看约束和数据集以尝试确定SQL语句的正确顺序。 true
tableOrdering 表名的列表，用于对DELETE操作进行重新排序，以防止由于循环依赖性而导致的失败。 “”
disableConstraints 禁止数据库约束。 false
cleanBefore 如果为true，则Database Rider将尝试通过使用表顺序和蛮力以一种聪明的方式在测试之前删除数据库。 false
cleanAfter 如果为true，则Database Rider将在测试后尝试通过使用表顺序和暴力破解以一种聪明的方式删除数据库。 false
transactional 如果为true，则将在测试之前启动事务，并在测试执行之后提交事务。 false
executeStatementsBefore 测试前要执行的jdbc语句列表。 {}
executeStatementsAfter 测试后要执行的jdbc语句列表。 {}
executeScriptsBefore 测试前要执行的sql脚本文件列表。请注意，sql文件中的命令必须用;分隔。 {}
executeScriptsAfter 测试后要执行的sql脚本文件列表。请注意，sql文件中的命令必须用;分隔。 {}
